# AI Data Explainer+

AI-powered data analytics application for non-technical business owners. Upload your CSV or Excel files and get instant business insights, recommendations, and interactive visualizations powered by AI.

## Features

- **File Upload**: Support for CSV and Excel files up to 10MB
- **Automatic Analysis**: Statistical analysis and data type detection
- **AI-Powered Insights**: Business insights generated by AI
- **Interactive Charts**: Bar charts, pie charts, histograms, heatmaps
- **Chat with Data**: Ask questions about your data in natural language
- **Smart Recommendations**: Actionable business recommendations
- **PDF Reports**: Generate comprehensive analysis reports
- **Multiple AI Providers**: Ollama (local), Groq, OpenRouter with automatic fallback
- **Real-time Streaming**: Instant AI responses with Server-Sent Events

## Tech Stack

### Frontend
- HTML5, CSS3, Vanilla JavaScript
- Chart.js for visualizations

- Node.js, Express.js

### AI Providers
- Ollama (local LLM)
- Groq API
- OpenRouter API

## Getting Started

### Prerequisites

- Node.js (v18 or higher)
- npm or yarn
- Docker (optional)
- Ollama (for local AI provider)

### Installation

1. **Clone the repository**
   ```bash
   git clone <repository-url>
   cd ai-data-explainer
   ```

2. **Install dependencies**
   ```bash
   cd backend
   npm install
   ```

3. **Configure environment variables**
   ```bash
   cp .env.example .env
   ```
   
   Edit `.env` with your configuration:
   ```env
   PORT=3000
   NODE_ENV=development
   AI_PROVIDER=ollama
   OLLAMA_URL=http://localhost:11434
   OLLAMA_MODEL=llama3.2
   GROQ_API_KEY=your_groq_api_key_here
   OPENROUTER_API_KEY=your_openrouter_api_key_here
   ```

4. **Start Ollama (if using local AI)**
   ```bash
   # Install Ollama
   # macOS: brew install ollama
   # Linux: curl -fsSL https://ollama.com/install.sh | sh
   # Windows: Download from https://ollama.com/download
   
   # Pull a model
   ollama pull llama3.2
   
   # Start Ollama
   ollama serve
   ```

5. **Start the backend server**
   ```bash
   npm start
   ```
   
   For development with auto-reload:
   ```bash
   npm run dev
   ```

6. **Open the application**
   - Open `frontend/index.html` in your browser
   - Or serve it with a simple HTTP server:
     ```bash
     cd frontend
     python -m http.server 8080
     ```

## Usage

### 1. Upload Data
- Click "Upload File" or drag and drop your CSV/Excel file
- Maximum file size: 10MB
- Supported formats: CSV, Excel (.xlsx)

### 2. View Analysis
- Automatic statistical analysis
- Data preview table
- Column type detection
- Missing values analysis

### 3. Generate Charts
- Automatic chart generation
- Bar charts for categorical data
- Pie charts for distribution
- Histograms for numeric data
- Heatmaps for correlations

### 4. Get AI Insights
- Click "Generate Insights"
- AI analyzes your data
- Executive summary and key findings
- Patterns, trends, and outliers

### 5. Chat with Data
- Ask questions in natural language
- Get instant AI responses
- Streaming responses for real-time feedback
- Conversation history maintained

### 6. Get Recommendations
- Click "Generate Recommendations"
- AI provides actionable suggestions
- Priority-based recommendations
- Expected impact analysis

### 7. Generate Report
- Click "Generate PDF Report"
- Comprehensive analysis report
- Includes all insights and recommendations
- Download as PDF

## API Endpoints

### Health Check
- `GET /api/health` - Server health status

### File Upload
- `POST /api/upload` - Upload CSV/Excel file

### Analysis
- `POST /api/analyze` - Analyze dataset

### Insights
- `POST /api/insights` - Generate AI insights

### Chat
- `POST /api/chat` - Chat with data (streaming)
- `GET /api/chat/history/:sessionId` - Get chat history
- `DELETE /api/chat/history/:sessionId` - Clear chat history

### Recommendations
- `POST /api/recommendations` - Generate recommendations

### Report
- `POST /api/report` - Generate PDF report

### Provider Management
- `GET /api/provider/health` - Check AI provider health
- `GET /api/provider/stats` - Get provider statistics
- `POST /api/provider/switch` - Switch AI provider

### Performance
- `GET /api/performance/metrics` - Get performance metrics
- `GET /api/performance/system` - Get system metrics
- `GET /api/performance/alerts` - Get performance alerts

## Configuration

### Environment Variables

| Variable | Description | Default |
|----------|-------------|---------|
| PORT | Server port | 3000 |
| NODE_ENV | Environment | development |
| AI_PROVIDER | Primary AI provider | ollama |
| OLLAMA_URL | Ollama API URL | http://localhost:11434 |
| OLLAMA_MODEL | Ollama model | llama3.2 |
| GROQ_API_KEY | Groq API key | - |
| OPENROUTER_API_KEY | OpenRouter API key | - |
| MAX_FILE_SIZE | Max file size in bytes | 10485760 |
| UPLOAD_DIR | Upload directory | ./backend/uploads |
| REPORT_DIR | Report directory | ./backend/reports |
| CORS_ORIGIN | CORS origin | * |
